FINAL PROJECT PROMPT

Build a cinematic developer portfolio using Next.js and React Three Fiber.

The goal is a memorable but recruiter-friendly experience, not a game.

General Requirements

The portfolio must:

load quickly

remain usable for recruiters

allow deep linking to sections

provide a cinematic but lightweight 3D experience

Target desktop users primarily, with a simplified fallback for mobile.

Landing Page

The root route / renders a 3D forest hub scene.

The forest scene contains three visible paths:

Projects
Skills & Achievements
Education & Co-curricular

Each path should:

glow or brighten on hover

show a floating text label

slightly bias the camera toward the hovered path

highlight the trail using emissive lighting

When a user clicks a path:

input locks

camera performs a short dolly animation (~3 seconds)

screen fades

router navigates to the corresponding route

Scenes should be preloaded when hovering over a path.

Routes

The portfolio must support direct navigation to:

/
 /projects
 /skills-achievements
 /education-cocurricular

These routes should work even if users skip the landing page.

Destination Scenes

Each route contains a themed 3D environment.

Projects

Environment theme:

Waterfall valley

Contains 3 floating objects representing projects.

Examples:

stone tablets
wooden signs
ancient relics

Clicking an object opens a UI panel showing:

project title
description
tech stack
GitHub link
live demo link
Skills & Achievements

Environment theme:

Open grasslands

Contains 3 objects representing:

technical skills
major achievements
tools / technologies

Examples:

totems
banners
wind structures

Clicking reveals skill details.

Education & Co-curricular

Environment theme:

cave interior

Contains 3 objects representing:

education milestones
clubs
leadership
competitions

Examples:

crystals
carvings
lantern altars

Clicking reveals details.

Persistent UI

Each scene includes a minimal UI overlay with:

Back to Forest
Quick Navigation
Resume Download
Contact

UI should remain lightweight and readable.

Content System

Portfolio content must be data-driven.

Create a structured content file:

content/portfolio.ts

This file defines:

paths
scene themes
objects
panel content
links

Scene components must generate objects dynamically from this data.

Performance Requirements

The experience must remain smooth.

Guidelines:

lazy load scenes
preload hovered assets
limit model size
compress GLB models

Total scene complexity should remain low.

Mobile Fallback

Mobile devices should not render full 3D scenes.

Instead show a simplified portfolio page containing:

hero section
project cards
skills
education
contact
Accessibility

Support:

reduced motion
keyboard navigation
readable UI panels

Transitions should shorten for reduced motion users.
